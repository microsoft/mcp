# FileShares Live Test Re-Recording Plan

## Context
With the new API version in `Azure.ResourceManager.FileShares`, existing recorded test sessions are invalidated and must be re-captured.

## Prerequisites
- Azure CLI authenticated to TME Organization tenant
- Test resources deployed in `SSS3PT_ankushb-mcp7395a3c9` resource group

## Steps

### 1. Deploy Test Resources
```powershell
Connect-AzAccount -SubscriptionId 4d042dc6-fe17-4698-a23f-ec6a8d1e98f4 -TenantId 70a036f6-8e4d-4615-bad6-149c02e7720d
./eng/scripts/Deploy-TestResources.ps1 -Paths FileShares
```

### 2. Set TestMode to Record
Edit: `tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests/.testsettings.json`
Change: `"TestMode": "Playback"` → `"TestMode": "Record"`

### 3. Run Tests (Records new sessions)
```powershell
dotnet test tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests -p:TreatWarningsAsErrors=false
```

### 4. Inspect Recordings
```powershell
# Locate recording folder
./.proxy/Azure.Sdk.Tools.TestProxy.exe config locate -a tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests/assets.json
```
Manually review JSON files in `SessionRecords/` folder for secrets/unstable data.

### 5. Push Recordings to Azure SDK Assets
```powershell
./.proxy/Azure.Sdk.Tools.TestProxy.exe push -a tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests/assets.json
```

### 6. Set TestMode to Playback
Edit: `tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests/.testsettings.json`
Change: `"TestMode": "Record"` → `"TestMode": "Playback"`

### 7. Run Tests (Playback mode - uses recordings)
```powershell
dotnet test tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests -p:TreatWarningsAsErrors=false
```

### 8. Verify All Tests Pass
```powershell
dotnet test tools/Azure.Mcp.Tools.FileShares/tests/Azure.Mcp.Tools.FileShares.LiveTests -p:TreatWarningsAsErrors=false --logger "console;verbosity=minimal"
```

## Notes
- The controlling `.testsettings.json` is in the LiveTests project directory (not the outer `tests/` directory)
- The LiveTests config uses `AzureCliCredential` — ensure `az login` points to the correct tenant (`70a036f6-...`)
- The outer `tests/.testsettings.json` (generated by deploy script) uses `AzurePowerShellCredential` and is not used by the test proxy
- After pushing recordings, commit the updated `assets.json` (new Tag). `.testsettings.json` is gitignored.
- The CRUD test creates then deletes a share named `{baseName}-crud` — needs write permissions
- Use `-p:TreatWarningsAsErrors=false` to bypass transient NuGet vulnerability warnings (NU1902/NU1903/NU1904)
